What it does

Aware of your repo. Silent by default.

Ticks every 5 minutes, reads everything, speaks only when it has something useful to say.

Repo awareness

Reads git history, modified files, TODOs, FIXMEs, HACKs, and missing env keys on every tick.

Presence detection

Detects if you're at your Mac via idle time. Loud when you're away, quiet when you're present.

Autonomous PR comments

Finds something relevant to an open PR? It posts a comment autonomously, prefixed with KAIROS:.

Persistent memory

Every night consolidates observations into a rolling MEMORY.md so it gets smarter over time.

Desktop notifications

Step away for 5 minutes? You'll get a native Mac notification with whatever it found.

Never duplicates

Logs every action to SQLite. Won't surface the same nudge twice in a day.

Architecture

How the tick loop works

Every minute: build context → detect presence → ask the model → act or sleep.

Setup

Up in 2 minutes

Requires Python 3.10+, a Groq API key, and a GitHub personal access token.

1

Clone and install

git clone https://github.com/nandanadileep/Kairos-mini.git
cd Kairos-mini
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
2

Configure your .env

Copy the example and fill in your keys.

cp .env.example .env
KeyDescription
GITHUB_TOKENrequiredPersonal access token with repo scope
GITHUB_REPOrequiredRepo to watch, e.g. username/repo
GROQ_API_KEYrequiredFrom console.groq.com — free tier works
GOOGLE_AI_STUDIO_KEYrequiredFrom aistudio.google.com — for tick model
KAIROS_REPO_PATHrequiredAbsolute path to your local repo
OLLAMA_BASE_URLoptionalDefaults to http://localhost:11434
3

Run

python3 main.py

KAIROS starts silently. It prints a line only when it has something useful to say.
